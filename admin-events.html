⚙️ Configuration — 2 clics

Il faut créer la table events dans Supabase une seule fois. Suis les 2 étapes ci-dessous.

1

Copie le SQL

-- Table des événements
CREATE TABLE IF NOT EXISTS events (
  id             uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  titre          text NOT NULL,
  date_event     timestamptz NOT NULL,
  description    text,
  image_url      text,
  brevo_form_url text,
  actif          boolean DEFAULT false,
  created_at     timestamptz DEFAULT now()
);
ALTER TABLE events ENABLE ROW LEVEL SECURITY;
CREATE POLICY "public_read_events"
  ON events FOR SELECT USING (true);
CREATE POLICY "admin_write_events"
  ON events FOR ALL
  USING (auth.jwt() ->> 'email' = 'tallecbastian.pro@gmail.com');
2

Ouvre l'éditeur SQL Supabase → colle → clique Run

Ça ouvre directement ton projet. Ctrl+V dans l'éditeur, puis le bouton Run en haut à droite.

🚀 Ouvrir l'éditeur Supabase →

✅ Une fois le script exécuté, reviens ici et crée ton premier événement dans l'onglet Événements.
La page publique event.html affichera automatiquement l'événement actif avec le formulaire Brevo en iframe.

ℹ️ Comment fonctionne l'intégration Brevo ?

Dans le champ URL formulaire Brevo, colle l'URL complète de ton formulaire hébergé (ex: https://2fabf66e.sibforms.com/serve/...).

La page event.html l'affiche en iframe — Brevo gère tout : inscription, email de confirmation, base de contacts.

Le QR code (onglet QR Code) pointe directement vers cette URL pour une utilisation en présentiel.
